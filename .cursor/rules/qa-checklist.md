# Чек-лист контроля качества кода (QA & Code Review)

Перед тем как выдать финальный ответ и сделать запись в дневник, ИИ-агент обязан проверить код по этому списку:

- [ ] **Соответствие стеку:** В коде точно нет упоминаний Inertia, Livewire или Filament?
- [ ] **Баланс KISS/YAGNI:** Не создан ли лишний пустой класс-прокладка (`Service`/`Action`/`DTO`) ради 1-2 строк CRUD-кода?
- [ ] **Баланс SOLID/DRY:** Если код объемный (>12 строк) или дублируется, вынесен ли он из контроллера в Сервис/Scope?
- [ ] **Классы final:** Все новые `Services`, `Actions` и `DTO` объявлены как `final`?
- [ ] **Типизация:** Добавлен ли `declare(strict_types=1);` и везде ли проставлены строгие типы данных?
- [ ] **Интеграция с Boost:** Была ли проверена структура базы данных или вызов кода через Tinker (Laravel Boost) перед коммитом?
- [ ] **Локализация:** Все ли текстовые строки интерфейса обернуты в хелперы i18n (`__('...')`)?
- [ ] **Тестируемость:** Написан ли соответствующий Pest/PHPUnit тест (Unit для изолированной логики/DTO, Feature для HTTP/БД цепочек)?
